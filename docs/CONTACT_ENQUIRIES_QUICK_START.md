# Contact Enquiries - Quick Start Guide

## üöÄ Accessing the Feature

1. Log into the Royal Drive Admin Panel
2. Navigate to **Procurement** ‚Üí **Contact Enquiries** in the sidebar
3. You'll see the enquiries list with filtering options

## üìù Common Tasks

### View All New Enquiries
```
1. Go to Contact Enquiries page
2. Select "New" from Status filter
3. Review the list of new submissions
```

### Respond to an Enquiry
```
1. Click "View" on the enquiry
2. Update Status to "Contacted"
3. Add a note: "Called customer, discussed requirements"
4. Log contact in Contact History:
   - Method: Phone
   - Notes: "Customer interested in financing options"
5. Update Priority if needed
```

### Mark Enquiry as Resolved
```
1. Open the enquiry detail page
2. Verify all actions are complete
3. Click "Mark Resolved" button
4. System automatically logs resolution date and user
```

### Search for Specific Customer
```
1. Use search box to enter:
   - Customer name
   - Email address
   - Phone number
   - Keywords from message
2. Results filter in real-time
```

### Filter by Priority
```
1. Click Priority dropdown
2. Select: Low, Medium, High, or Urgent
3. View filtered results
```

## üé® Status Meanings

| Status | When to Use |
|--------|-------------|
| **New** | Enquiry just received, not yet reviewed |
| **Contacted** | Customer has been reached out to |
| **In Progress** | Actively working on the request |
| **Resolved** | Issue resolved, customer satisfied |
| **Closed** | Enquiry closed without resolution |

## ‚ö° Priority Levels

| Priority | When to Use |
|----------|-------------|
| **Low** | General inquiries, no urgency |
| **Medium** | Standard customer requests (default) |
| **High** | Time-sensitive or important customer |
| **Urgent** | Critical issues requiring immediate attention |

## üìä Understanding the Dashboard Card

The Contact Enquiries stats card shows:
- **Total**: All enquiries in the system
- **New**: Unreviewed enquiries (needs attention)
- **Status Breakdown**: Distribution across all statuses
- **Priority Counts**: How enquiries are prioritized
- **Recent Enquiries**: Last 3 submissions

## üîî Best Practices

### 1. **Daily Review**
Check new enquiries every morning:
```
- Filter by Status: "New"
- Sort by: "Newest First"
- Review and prioritize
```

### 2. **Quick Triage**
For each new enquiry:
```
1. Read customer message
2. Set appropriate priority
3. Update status to "Contacted" or "In Progress"
4. Assign to team member if needed
```

### 3. **Keep Notes Updated**
Add notes when:
```
- Customer calls back
- You need to follow up later
- Special requirements are mentioned
- Other team members need context
```

### 4. **Log All Contact**
Record every interaction:
```
Phone Call ‚Üí Log in Contact History
Email Sent ‚Üí Log in Contact History
Meeting Scheduled ‚Üí Add Note + Log Contact
```

### 5. **Regular Cleanup**
Weekly maintenance:
```
- Mark resolved enquiries as "Closed"
- Follow up on "In Progress" items
- Review high priority items
```

## üÜò Troubleshooting

### "Failed to fetch enquiries"
- Check your internet connection
- Verify you're logged in
- Contact IT if problem persists

### Can't update status
- Ensure you have Admin permissions
- Refresh the page and try again
- Check if enquiry is already resolved

### Missing enquiries
- Clear active filters (click "Clear All")
- Check if you're on the correct page number
- Verify enquiries exist in the system
